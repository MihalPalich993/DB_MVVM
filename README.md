# DB_MVVM — CRM-система торговой компании

Курсовая работа по дисциплине **«Технология разработки программного обеспечения»**.  
Специальность: **09.02.07 Информационные системы и программирование**.

## 📌 Общие сведения
Приложение предназначено для автоматизации учета клиентов и управления историей заказов в торговой компании. Проект реализован с акцентом на разделение ответственности слоев (MVVM), безопасность данных и удобство пользовательского интерфейса (UX/UI).

## 🛠 Технологический стек
*   **Платформа:** .NET 8 (LTS)
*   **Интерфейс:** WPF (Windows Presentation Foundation)
*   **Архитектура:** паттерн **MVVM** (использована библиотека `CommunityToolkit.Mvvm`)
*   **База данных:** SQLite (встраиваемая, не требует настройки сервера)
*   **ORM:** Entity Framework Core (подход Code First)

## 🚀 Ключевой функционал
*   **Система ролей (RBAC):** 
    *   `Admin` — полный доступ, удаление записей, просмотр журнала аудита.
    *   `Manager` — создание и редактирование клиентов/заказов (удаление запрещено).
*   **Автоматизированный Аудит:** реализовано логирование всех действий пользователей в реальном времени через перехват событий в `DbContext`.
*   **Интерактивная аналитика (Dashboard):** автоматический расчет общего числа клиентов, заказов и выручки на главной панели.
*   **Умная валидация:** проверка Email (запрет кириллицы через Regex), телефонов и сумм заказов с визуальной индикацией ошибок.
*   **Master-Detail интерфейс:** мгновенное отображение заказов при выборе клиента в таблице.
*   **Живой поиск:** фильтрация базы клиентов по фамилии в процессе ввода текста.

## 📐 Архитектура программного решения
Проект организован по модульному принципу:
*   `Models` — классы сущностей и логика валидации (`ObservableValidator`).
*   `ViewModels` — разделенная логика: `MainViewModel`, `ClientsViewModel`, `OrdersViewModel` и `LogViewModel`.
*   `Views` — XAML-разметка окон и глобальные стили приложения.
*   `Data` — контекст базы данных и конфигурация SQLite.

## 📦 Установка и запуск
1. Склонируйте репозиторий.
2. Откройте файл `DB_MVVM.sln` в Visual Studio 2022.
3. Соберите решение (**Build**). 
4. База данных `TradingCompany.db` создастся автоматически при первом запуске программы.
